# Roadmap: Dependency Graph Viewer

## Overview

Starting from the scaffolded Next.js 16 + React 19 + @xyflow/react baseline, this roadmap delivers a working dependency graph viewer in four required phases plus one polish phase. Phase 1 establishes the static graph canvas with correct data flow and layout, resolving all known rendering pitfalls before anything is interactive. Phase 2 wires SPOF detection into always-visible node decoration. Phase 3 adds the full selection and info sidebar experience. Phase 4 completes the required controls (reload, sort toggles). Phase 5 adds the senior-level differentiators that distinguish this submission.

## Phase Details

### Phase 1: Static Graph Render
**Goal**: Users can see all 23 nodes and 30 edges rendered in a zoomable, pannable canvas with activity and resource nodes visually distinct and edges showing dependency direction
**Depends on**: Nothing (first phase)
**Requirements**: GRAPH-01, GRAPH-02, GRAPH-03, GRAPH-04
**Success Criteria** (what must be TRUE):
  1. All 8 activity nodes and 15 resource nodes from graph.json appear on the canvas
  2. Activity nodes and resource nodes are visually distinct (different color or shape)
  3. Edges have arrowheads showing direction from activity to resource
  4. Nodes are positioned in a hierarchical dagre layout (not stacked at 0,0)
  5. Canvas supports zoom and pan without blank/invisible rendering

- [x] 01-02-PLAN.md — Wave 2: full implementation — types, transform, node components, dagre layout hook, GraphCanvas, page wiring

### Phase 3: Selection and Info Panel
**Goal**: Users can click any node to understand exactly what depends on what, with non-connected elements dimmed and a sidebar showing full node details
**Depends on**: Phase 2
**Requirements**: INTR-01, INTR-02, INTR-03, INTR-04, INTR-05, PANEL-01, PANEL-02, PANEL-03, PANEL-04
**Success Criteria** (what must be TRUE):
  1. Clicking a node highlights it and its immediate connections; all other nodes and edges are visually dimmed
  2. Clicking the canvas background restores full visibility for all nodes and edges
  3. Clicking the already-selected node again clears the selection and restores full visibility
  4. A fixed sidebar info panel appears on selection showing node name, type, and all dataset fields
  5. For SPOF resource nodes the sidebar shows a risk statement indicating how many activities depend on the resource
  6. The sidebar closes or returns to empty state when selection is cleared
**Plans**: 1 plan
Plans:
- [x] 03-01-PLAN.md — Wave 1: node selected rings + GraphCanvas selection state + opacity logic + InfoPanel component
**UI hint**: yes

### Phase 4: Toolbar Controls
**Goal**: Users can reload the graph from source data and reorder nodes by criticality using toolbar controls
**Depends on**: Phase 3
**Requirements**: CTRL-01, CTRL-02, CTRL-03
**Success Criteria** (what must be TRUE):
  1. A reload button re-fetches graph.json and re-renders the graph, resetting all selection and sort state
  2. An activities sort toggle reorders activity nodes by criticality (critical → high → medium → low)
  3. A resources sort toggle reorders resource nodes by criticality (SPOF first, then by dependency count descending)
