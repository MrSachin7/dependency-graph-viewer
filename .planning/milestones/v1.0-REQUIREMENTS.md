# Requirements: Dependency Graph Viewer

**Defined:** 2026-03-29
**Core Value:** A user can immediately see which resources are SPOFs and, by clicking any node, understand exactly what depends on what.

## v1 Requirements

### Graph Rendering

- [x] **GRAPH-01**: Graph renders all activity and resource nodes from graph.json as directed edges
- [x] **GRAPH-02**: Activity nodes and resource nodes are visually distinct (different shape, color, or label style)
- [x] **GRAPH-03**: Edges are directed (arrows show dependency direction: activity → resource)
- [x] **GRAPH-04**: Initial layout is computed automatically using dagre (@dagrejs/dagre) in a hierarchical arrangement
- [x] **GRAPH-05**: Resource type is visually indicated per node (technology, people, building, third_party, equipment — via icon or label badge)

### SPOF Detection

- [x] **SPOF-01**: Any resource node depended on by more than one node is computed as a Single Point of Failure
- [x] **SPOF-02**: SPOF nodes are visually distinct at a glance via a persistent badge showing dependency count and a ring/border color
- [x] **SPOF-03**: SPOF indicator is not tooltip-only — it is always visible on the node

### Interaction

- [x] **INTR-01**: Clicking a node highlights the node itself, its edges, and connected nodes
- [x] **INTR-02**: All non-connected nodes and edges are visually de-emphasized (dimmed) when a selection is active
- [x] **INTR-03**: Clicking the canvas background clears the selection and restores full visibility
- [x] **INTR-04**: Clicking the already-selected node again clears the selection
- [x] **INTR-05**: A fixed sidebar info panel appears on the right when a node is selected

### Info Panel

- [x] **PANEL-01**: Info panel displays node name, type, and all fields from the dataset for the selected node
- [x] **PANEL-02**: For resource nodes, the panel includes a SPOF indicator if the node is a SPOF
- [x] **PANEL-03**: For SPOF resource nodes, the panel shows a risk statement: how many activities depend on it
- [x] **PANEL-04**: Panel closes (or returns to empty state) when selection is cleared

### Controls

- [x] **CTRL-01**: A reload button re-fetches graph.json and re-renders the graph
- [x] **CTRL-02**: An activities sort toggle orders activity nodes by criticality (critical → high → medium → low)
- [x] **CTRL-03**: A resources sort toggle orders resource nodes by criticality (SPOF first, then by dependency count descending)

## v2 Requirements

### Enhancements

- **V2-01**: Edge color or style varies by relationship type
- **V2-02**: RTO value displayed on activity nodes
- **V2-03**: Zoom to fit on load and after reload
- **V2-04**: Keyboard navigation (Escape to deselect)
- **V2-05**: Export graph as PNG

## Out of Scope

| Feature | Reason |
|---------|--------|
| Backend / API | Assignment specifies static JSON only |
| Authentication | Not required for take-home |
| Mobile responsiveness | Desktop-only viewport assumed |
| Pixel-perfect design | Assignment explicitly states functional and clear is enough |
| Automatic layout perfection | Assignment explicitly states reasonable initial layout is fine |
| Real-time data | No backend; reload button covers data refresh |
