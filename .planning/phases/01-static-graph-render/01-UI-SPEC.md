# Phase 1 — UI Design Contract: Static Graph Render

> Visual and interaction contract for Phase 1. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> Phase 1 is read-only: no user interaction, no selection state, no sidebar. The canvas renders all 23 nodes and 29 edges in a zoomable/pannable dagre layout with activity vs. resource nodes visually distinct.

---

| Property | Value |

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, handle padding inside node |
| sm | 8px | Node label inner padding (top/bottom) |
| md | 16px | Node label inner padding (left/right), node width buffer |
| lg | 24px | dagre node separation (rankSep horizontal gap) |
| xl | 32px | dagre rank separation (vertical layer gap between activity row and resource row) |
| 2xl | 48px | Canvas outer padding (fitView padding factor translates to ~48px visual bleed) |
| 3xl | 64px | Minimum dagre nodeSep — horizontal distance between nodes in the same rank |

Exceptions:
- Node dimensions: activity nodes 160px wide × 48px tall; resource nodes 160px wide × 44px tall. These are fixed measurement anchors required by dagre layout, not spacing tokens.
- React Flow canvas: `width: 100%; height: 100vh` — fills the full viewport, no page-level padding applied to the canvas container.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#f6f4ee` | Full-page background, React Flow canvas background |
| Secondary (30%) | `#ffffff` | Node fill for both activity and resource nodes |
| Accent — Activity (10%) | `#2563eb` (blue-600) | Activity node border (2px solid), activity node label text |
| Accent — Resource | `#16a34a` (green-600) | Resource node border (2px solid), resource node label text |
| Foreground text | `#171717` | Default body text, fallback node label if accent not applied |
| Edge stroke | `#64748b` (slate-500) | All edge lines and arrowheads |
| Canvas dots | `#d1d5db` (gray-300) | Background dot pattern via React Flow `<Background>` component |
| Destructive | not used in Phase 1 | — |

Accent reserved for:
- Activity node border color: `#2563eb`
- Resource node border color: `#16a34a`
- Node label text color mirrors its node's border color (activity = blue, resource = green)

Color rationale: The warm off-white `#f6f4ee` is the established scaffold background. White node fills on this surface create readable contrast. Blue/green split gives immediate visual distinction between the two node types without hover or selection state, satisfying GRAPH-02.

Source: `--background: #f6f4ee` from `globals.css`; blue/green node colors are default specification (no prior source).

## Node Visual Specification

**Visual Hierarchy:** Primary focal point is the activity node row (top rank, blue border). The eye enters the graph at the blue nodes and follows edges downward to the green resource nodes in the lower rank.

### Activity Node (`type: "activity"`)

| Property | Value |
|----------|-------|
| Width | 160px |
| Height | 48px |
| Border | 2px solid `#2563eb` |
| Border radius | 8px |
| Background | `#ffffff` |
| Label color | `#2563eb` |
| Label size | 13px, weight 500 |
| Label position | Centered horizontally and vertically |
| Handle (source) | Bottom center — activity is the dependency source |
| Handle (target) | None — activities have no incoming edges in Phase 1 data (act-8 → act-6 is the exception; act-6 needs a target handle) |

Note on act-8 → act-6 edge: act-6 appears as a target in one edge (`"from": "act-8", "to": "act-6"`). Activity nodes must have both source and target handles; dagre will position act-6 in a rank where it receives an incoming edge from act-8.

### Resource Node (`type: "resource"`)

| Property | Value |
|----------|-------|
| Width | 160px |
| Height | 44px |
| Border | 2px solid `#16a34a` |
| Border radius | 6px (slightly less rounded than activity to aid distinction) |
| Background | `#ffffff` |
| Label color | `#16a34a` |
| Label size | 13px, weight 500 |
| Label position | Centered horizontally and vertically |
| Handle (source) | None — resources are leaf nodes (no outgoing edges except via act-8 → act-6 case, which is activity-to-activity) |
| Handle (target) | Top center — resources receive incoming dependency edges |

### Edges

| Property | Value |
|----------|-------|
| Edge type | `smoothstep` (React Flow built-in) |
| Stroke color | `#64748b` |
| Stroke width | 1.5px |
| Arrowhead | `markerEnd: MarkerType.ArrowClosed` (React Flow built-in) |
| Arrow fill | `#64748b` |
| Animated | false (static render) |

Source: React Flow @xyflow/react skill — `getSmoothStepPath`, `MarkerType.ArrowClosed`.

### Canvas

| Property | Value |
|----------|-------|
| Background | `BackgroundVariant.Dots`, color `#d1d5db`, gap 20, size 1 |
| Controls | `<Controls />` component from @xyflow/react (zoom in, zoom out, fit view, lock) — bottom-left default position |
| MiniMap | Omitted in Phase 1 (adds visual noise before SPOF coloring is in place) |
| fitView | true — on initial mount, padding 0.1 |

## Layout: dagre Configuration

| Property | Value |
|----------|-------|
| Direction | `TB` (top-to-bottom) — activities row above, resources row below |
| rankSep | 80 — vertical gap between ranks |
| nodeSep | 40 — horizontal gap between nodes in same rank |
| Node width (dagre) | 160px (matches rendered node width) |
| Node height (dagre) | 48px (activity) / 44px (resource) — use 48px for both in dagre for uniform rank spacing |
| Layout trigger | Gated on `useNodesInitialized() === true` (see STATE.md Phase 1 caution) |

dagre package: `@dagrejs/dagre`. Must be installed if not already present.

## Interaction Contract (Phase 1 Scope)

Phase 1 is passive display only. The following interactions are provided by React Flow defaults and require no custom implementation:

| Interaction | Behavior |
|-------------|----------|
| Scroll / trackpad pinch | Zoom in and out |
| Click and drag canvas | Pan the viewport |
| Click and drag node | Move node (default behavior — not disabled in Phase 1) |
| Double-click canvas | No action (React Flow default) |
| Node click | No action (selection wired in Phase 3) |

No `onNodeClick`, `onEdgeClick`, or `onPaneClick` handlers are registered in Phase 1.

## Component Inventory (Phase 1)

These are the components the executor must create:

| Component | Path | Type | Notes |
|-----------|------|------|-------|
| `GraphPage` | `src/app/page.tsx` | Client Component (`"use client"`) | Root page; wraps ReactFlowProvider, loads graph.json, passes nodes/edges to GraphCanvas |
| `GraphCanvas` | `src/components/GraphCanvas.tsx` | Client Component | Renders `<ReactFlow>` with nodeTypes, edgeTypes, dagre layout, Background, Controls |
| `ActivityNode` | `src/components/nodes/ActivityNode.tsx` | Memo component | Custom node for `type: "activity"` — blue border, top/bottom handles |
| `ResourceNode` | `src/components/nodes/ResourceNode.tsx` | Memo component | Custom node for `type: "resource"` — green border, target handle only |
| `useGraphData` | `src/hooks/useGraphData.ts` | Hook | Loads graph.json, transforms raw nodes/dependencies into React Flow `Node[]` and `Edge[]` |
| `useDagreLayout` | `src/hooks/useDagreLayout.ts` | Hook | Runs dagre on initialized nodes, returns positioned Node[] |
| `graph.types.ts` | `src/types/graph.types.ts` | Types | `ActivityNodeData`, `ResourceNodeData`, `GraphJson`, `GraphNode`, `GraphEdge` |

`nodeTypes` and `edgeTypes` objects must be defined at module scope in `GraphCanvas.tsx` (not inside the component body) to prevent all 23 nodes remounting on every render. Source: STATE.md Phase 1 caution.
